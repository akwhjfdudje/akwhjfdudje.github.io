Hello. This post is about recursive descent.

Parsing

We will start with some background. Compilers, for a long time now, have been made with a particular design in mind. Said design involves the stages of lexing, parsing, analysis, and code generation.

Note some things at this point:

  • This is not the only way to read programming languages.
  • The stage of "analysis" nowadays contains several "substages", which are not important for now.
  • "Code" in "code generation" here refers to the direct assembly that a compiled language turns into. This is, of course, not the only output of a compiler. There are certainly several more "code"s that a compiler may output (that are not pure assembly).

The stage of lexing will be left as a black box for now. All that is important is that lexing turns a sequence of words, symbols, etc. into a set of "tokens".

The stage of parsing takes the set of "tokens" from lexing, and uses an algorithm to give "meaning" to the "tokens". Which algorithm is chosen depends on the design of the parsing stage, the needs of the language developer, opinions, etc.

The topic of this post, recursive descent, is one of the aforementioned algorithms.
